Enable GSLB

Assuming you have an application that provides services to internet users through https://service.example.com, you can refer to the following steps to enable GSLB (Global Server Load Balance) if you have multiple gateway nodes deployed.

  1. Step 1: Enable firewall policy, example for Debian 11: #ufw allow 53
  2. Step 2: Make sure port 53 is not occupied by other applications, typically in Debian 11: #systemctl stop systemd-resolved and #systemctl disable systemd-resolved, then check with: #netstat -antulp | grep :53
  3. Step 3: Enable DNS Server in Settings - Advanced, and then restart service: #systemctl restart janusec
  4. Step 4: Add an A or CNAME record ns01 with value ip address points to this gateway, and a NS record service with value ns01.example.com. at the authoritative name server, not on this page. Individual domain name holders should modify it at the domain name registrar.
  5. Step 5: Add an A record service on this page, and enable Automatically resolve to an available gateway node
  6. Step 6: Configure application in Application and make sure the backend source servers are available to all nodes of Janusec Application Gateway.
  7. Step 7: Open https://service.example.com with web browser, or nslookup service.example.com under command shell.
